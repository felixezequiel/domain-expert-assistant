# Architecture Decision Records

| #                                                 | Title                               | Decision (one-liner)                                                                                                                                                |
| ------------------------------------------------- | ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [001](001-native-type-stripping.md)               | Native Type Stripping               | Use SWC (`@swc-node/register`) instead of Node's native type stripping — needed to support legacy TS decorators required by MikroORM.                               |
| [002](002-tc39-decorators.md)                     | TC39 Decorators                     | Stay on legacy (experimental) decorators while MikroORM doesn't support TC39 stage-3 decorators.                                                                    |
| [003](003-domain-event-dispatch.md)               | Domain Event Dispatch               | Aggregates emit events; `ApplicationService` drains, dispatches in-process via `DomainEventManager`, then publishes externally — no pub/sub from inside the domain. |
| [004](004-unit-of-work-pattern.md)                | Unit of Work Pattern                | `TrackedUnitOfWork` + `AggregateTracker` (AsyncLocalStorage) — aggregates auto-track on first event, no manual `repository.save()` calls.                           |
| [005](005-application-service-orchestration.md)   | Application Service Orchestration   | Fixed orchestration order: `begin → execute → drain → dispatch → publish → saveAll(eventStore) → commit`.                                                           |
| [006](006-event-store-and-real-world-patterns.md) | Event Store and Real-World Patterns | All domain events persisted to `system_events` inside the aggregate transaction (no dual-write divergence) for audit trail and event-sourcing-readiness.            |
| [007](007-multi-assistant-rules-strategy.md)      | Multi-Assistant Rules Strategy      | `.claude/skills/` is the single source of truth; `scripts/sync-rules.mjs` generates adapters for Cursor, Windsurf, Cline, Continue, Aider — pre-commit is the safety net. |
| [008](008-actor-context-and-event-enrichment.md)  | Actor Context & Event Envelope Enrichment | Borda abre Actor Context `{companyId, actorId, actorType}` do principal autenticado; o `ApplicationService` carimba os eventos no drain e faz cross-check fail-closed `aggregate.companyId === contexto` para eventos de agregado — domínio fica puro. |
| [009](009-tenancy-isolation-guarantees.md)        | Tenancy Isolation Guarantees        | Isolamento em duas camadas fail-closed (filtro de leitura no UoW + trava de escrita da ADR-008); acesso sem tenant é erro, exceto escopo privilegiado explícito (operador/sistema); `actorType` = `user`/`consumer`/`system`/`operator`. **Emenda (ADR-018/022): RLS na v1 como piso de tenant.** |
| [010](010-authentication-and-credential-strategy.md) | Authentication & Credential Strategy | Sessão server-side opaca e revogável para humanos (não JWT — revogação imediata exigida por `DisableUser`); senha em `argon2id`; API key opaca = `keyPrefix` + `secretHash`, segredo exibido 1x. Resolução do principal abre o Actor Context da ADR-008. |
| [011](011-authorization-model.md)                  | Authorization Model                 | Autz grosseira por papel é transversal: `ApplicationService` invoca `AuthorizerPort` (do contexto Identity) antes do `execute`, não-contornável por adapter; `Role` no shared kernel; invariantes de negócio (revisor ≠ autor) são domínio, não autz; escopo do consumidor → PRD-5. |
| [012](012-knowledge-item-aggregate-and-versioning.md) | KnowledgeItem: Agregado & Versionamento | Versões são snapshots imutáveis append-only **fora** do agregado (não coleção filha hidratada); o agregado guarda ponteiro `publishedVersion` que governa o que **serve** — editar/revisar nunca tira a versão publicada do ar; rollback cria nova versão de rascunho sem mutar histórico. |
| [013](013-knowledge-governance-lifecycle.md)       | Knowledge Governance Lifecycle      | Transições são métodos de domínio guardados (estado inválido inalcançável); `Deprecated` serve sinalizado-desatualizado, `Archived` sai do ar; `requireSeparateReviewer` é injetada como parâmetro no `approve` (sem acoplar agregados); apagar coleção não-vazia é bloqueado por caso de uso cross-aggregate. |
| [014](014-taxonomy-system-vs-tenant-tags.md)       | Taxonomy: Tags de Sistema vs Tenant | Tabela `tags` unificada, filtro `company_id = :tenant OR scope = 'system'`; tags de sistema são referência imutável semeada (`company_id` null), não dado de tenant — não fere a ADR-009 (emendada). `Tag` é agregado próprio, `slug` único por índice; remover tag em uso é bloqueado. |
| [015](015-ingestion-processing-model.md)           | Ingestion Processing Model          | Upload retorna na hora; processamento async **in-process** com a tabela `ingestion_jobs` como fila (sem broker), recuperação de jobs presos no startup; worker abre o próprio Actor Context (`system`); parsers `pdf-parse`/`mammoth`/texto atrás de `Extractor` por mime. Chunking foi para a PRD-4. |
| [016](016-file-storage-port.md)                    | File Storage Port                   | Arquivo bruto atrás de `FileStoragePort` (fs local v1 → S3 depois); fica **fora** do filtro de DB, então o adapter faz o isolamento: `storageRef` escopado por `companyId` + validação fail-closed do Actor Context em toda leitura. Bytes nunca são embeddados; arquivo retido pós-extração. |
| [017](017-local-embedding-and-chunking.md)         | Local Embedding & Chunking          | **BGE-M3 fp16** (1024-dim, janela 8192, 100+ idiomas, MIT, sem prefixo) via `@huggingface/transformers` v3, atrás de `EmbedderPort`; fallback gte-multilingual-base. Chunking structure-aware + orçamento ~512 tokens + overlap, favorecendo precisão (v1 sem reranker). |
| [018](018-persistence-engine-and-vector-store.md)  | Persistence Engine & Vector Store   | Mudança fundacional: **Postgres para tudo** (domínio + event store + índice), `pgvector` para vetores + full-text do Postgres para léxico (híbrido in-DB). Troca o driver SQLite do template; perde zero-config mas destrava RLS como defesa em profundidade (ADR-009). |
| [019](019-hybrid-search-and-reranking.md)          | Hybrid Search & Reranking           | Híbrida via **RRF** in-DB (pgvector + full-text do Postgres), sem calibrar escalas. Reranking cross-encoder (`bge-reranker-v2-m3`) fica atrás de `RerankerPort` **desligado na v1** — consumidor é LLM + corpus curado tornam o ganho especulativo; ligar só sob golden-set. |
| [020](020-index-projection.md)                     | Index Projection                    | Indexação é **assíncrona e eventualmente consistente** (nunca dentro do commit de publicação); event store é a fila, worker in-process idempotente abre Actor Context `system`; índice derivado e reconstruível (`RebuildIndex`); `Deprecated` fica indexado com flag stale, `Archived` é removido. |
| [021](021-mcp-server-transport-auth-and-tools.md)  | MCP Server: Transport, Auth & Tools | Streamable HTTP; **API key como `Bearer`** (v1) → resolve credencial → Actor Context; OAuth 2.1 é Fase 2 (ChatGPT não é alvo v1). Tools = ações, resources = itens via template escopado (sem enumerar tudo); REST+MCP num `KnowledgeQueryFacade` único. |
| [022](022-consumer-scope-enforcement.md)           | Consumer Scope Enforcement          | Escopo do consumidor é **pré-filtro** na query (nunca pós), fail-closed: piso de tenant por **RLS**, escopo por-credencial (coleções + teto + servível) por pré-filtro de aplicação; escopo efetivo = credencial ∩ pedido (só estreita); alerta do *filtered ANN* do pgvector. |
| [023](023-frontend-stack-and-repo-boundary.md)     | Frontend Stack & Repo Boundary      | **SPA React+Vite em `/web`** (monorepo, fora do hexágono — cliente da porta REST), servida estática pelo monólito na mesma origem; auth por cookie `httpOnly` (não localStorage); REST only (GraphQL adiado); editor markdown; papel na UI é UX, autz é server-side. |
| [024](024-read-model-identity-resolution.md)       | Read-Model Identity Resolution      | Read models resolvem `userId → nome` **server-side** via `UserDirectoryPort` transversal (Identity provê o adapter, tenant-scoped; `NullUserDirectory` para quem não exibe nomes) — curator/auditor não podem ler o roster admin-only; auditoria enriquece o **ator**, mantém o agregado como id forense; `GET /invitations/:token` público (token = segredo) dá org+e-mail+papéis para a tela de aceite, fail-closed. |
